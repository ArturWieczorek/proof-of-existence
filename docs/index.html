How are these proofs created? (make your own)

A proof is just a normal Cardano transaction carrying a small metadata note under label 1718. Nothing is locked to this website: any tool that can attach that note can create a proof, and any tool can read it back. Only the hash goes on-chain - your document stays private.

The note format (Cardano transaction metadata, label 1718):

{
  "1718": {
    "h":    "<the document's SHA-256 hash, 64 hex chars>",
    "alg":  "SHA-256",
    "ts":   "<an ISO-8601 timestamp>",
    "name": "<an optional short label>"
  }
}

1. Easiest - the command-line tool. Download notary.jar (needs Java 21 installed) and run:

# keyless, no sign-up (default provider is Koios):
java -jar notary.jar submit myfile.pdf "my label" \
    --network preprod --signing-key my.sk --address addr_test1v...

# or via Blockfrost (needs a free project id from blockfrost.io):
java -jar notary.jar submit myfile.pdf \
    --backend blockfrost --blockfrost-project-id preprod... --network preprod \
    --signing-key my.sk --address addr_test1v...

# or against a local Yaci DevKit node (no third party at all):
java -jar notary.jar submit myfile.pdf --backend yaci --network testnet \
    --signing-key my.sk --address addr_test1v...

# hash / verify are fully local - no key, no network, no account:
java -jar notary.jar hash myfile.pdf
java -jar notary.jar verify myfile.pdf <hash>

Download notary.jar | source, docs and all options on GitHub

2. Any other tool. Attach the JSON above under metadata label 1718 to a transaction (with a wallet that supports metadata, or cardano-cli), submit it, and it becomes a proof this page can verify. The tool is a convenience, not a lock-in.

The explorer link proves the record exists on-chain; only the local re-hash proves your file matches it. A proof shows a document existed no later than its block time, unchanged - not who authored it. Proof of Existence.
